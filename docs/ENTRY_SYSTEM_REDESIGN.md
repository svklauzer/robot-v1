# Редизайн системы входов и сопровождения

**Статус:** черновик на утверждение (код не трогаем, пока Капитан не одобрит)
**Дата:** 2026-06-15
**Контекст:** живой paper на HTX, 7 символов, dual-mode (scalp/range/CRT/trend), depth-движок на WS.

---

## 0. TL;DR

Сейчас «скальп» — это на самом деле **свинг-фейд на часовом коридоре**, надетый на ярлык scalp. Поэтому он шортит пробои (дерётся с трендом), а часовой ТФ для скальпера — концептуально неверный инструмент. Депт-движок (1m поток ордеров) — родной инструмент скальпера — построен, но «скальп» на него не опирается.

**Решение — разделить два понятия:**
- **Движок входа** (что генерирует сигнал) — у каждого свои ТФ и условие.
- **Профиль ведения** (как сайзим/держим/выходим) — scalp (быстро/мелко) или position (медленно/крупно).

Каждый движок **читает только свои ТФ** → не лезет против шерсти by design, без костыльных фильтров. Конфиг сворачивается в профили (убираем кашу из env).

---

## 1. Что не так сейчас (по живым данным 13–15 июня)

1. **«Scalp/range» мислейбл.** Берёт 1h-коридор, шортит сопротивление = mean-reversion свинг на 1h, не скальп. На пробое (BTC 64→65.7k, SOL 67→71, DOT 0.96→1.00) фейдит силу и ловит стопы. XRP −2.32, SOL −1.08 — всё шорты против ралли.
2. **Мы блокируем правильные сделки.** BTC/SOL `trend_up_candidate` (setup 78–100) → отклонены (`tp1_net_pnl_below_min_usdt`, спред). Лонги по тренду не проходят, а шорты против тренда — проходят. Асимметрия = слив рынку.
3. **CVD режет победителей** (BTC +0.08 из пика +0.54; XRP +0.52 из +1.11) — выход чувствителен к шуму потока.
4. **ТФ перемешаны.** Скальп смотрит на 1h. Тренд и range делят 4h-regime. Нет чёткой привязки «тип сделки ↔ свой ТФ».
5. **Конфиг — каша.** `PROD_GATE_*` ×6, `SYMBOL_PERF_*` ×5, `ANTI_DRAIN_*`, `OB_*`, `SCALP_*`, `POST_LOSS_*` в env + десятки в config.py. Нет структуры «по типу».

---

## 2. Принципы

1. **Когерентность ТФ.** Биас, вход и сопровождение каждого движка живут в одной ТФ-полосе. Движок не использует ТФ вне своей полосы для своего решения.
2. **Не драться с шерстью by design.** Не костыль-фильтр, а: в тренде range-фейд ВЫКЛЮЧЕН, трендовый движок берёт движение; скальп изолирован на LTF и идёт по потоку.
3. **Разделение «движок входа» и «профиль ведения».** Один профиль ведения может обслуживать несколько движков.
4. **Скальп = много/мелко/быстро; позиция = мало/крупно/едем.** Разные цели → разные таргеты и сопровождение.
5. **Конфиг — в профили, не в env.** Env только: секреты, инфра, HTX, telegram/billing, equity, режим, ON/OFF стратегий.
6. **Догма: плечо OFF**, включаем только на доказанном edge (не в этом редизайне).

## 3. Модель: Движки входа × Профили ведения

```
ДВИЖКИ ВХОДА (signal source)        ПРОФИЛЬ ВЕДЕНИЯ (sizing/hold/exit)
─────────────────────────          ─────────────────────────────────
micro-flow scalp   ── 1m/5m  ─────► SCALP   (мелко, быстро, тугой выход)
range mean-revert  ── 1h     ─────► POSITION (среднее, фейд края, до mid)
trend continuation ── 4h+15m ─────► POSITION (крупно, едем до структуры)
CRT (A→M→D)        ── 4h+5m  ─────► POSITION (едем до противоположной ликвидности)
```

Движок объявляет: свои ТФ, условие входа, направление-источник, профиль ведения.

## 4. Профили ведения

### SCALP profile
- **Сайзинг:** маленький (≈10% эквити кап), риск-based, мягкие абсолютные флоры.
- **Таргет:** мелкий — TP1 ≈ +0.3–0.6% net (достижимый, не «середина далёкого коридора»).
- **Стоп:** тугой (≈0.3–0.5%), за микроструктурой.
- **Сопровождение:** быстрый безубыток (после +0.2%), CVD-выход при развороте потока, **тайм-стоп** (не разрешилось за N минут → закрыть). Цикл 10с.
- **Частота:** высокая.

### POSITION profile
- **Сайзинг:** по конвикции (плечо OFF), но так, чтобы сделка реально открывалась (текущая проблема: тренд-лонг блокировался сайзингом).
- **Таргет:** крупный — до структуры/противоположной ликвидности, TP1 партиал + TP2.
- **Стоп:** по структуре (за свечой C2 для CRT, за свингом для тренда).
- **Сопровождение:** trend-ride (широкий трейл, держим пока MFE растёт; выход при отдаче доли пика). CVD НЕ форс-выходит (только как ускоритель трейла при сильном развороте).
- **Частота:** низкая.

## 5. Движки входа

### 5.1 Micro-flow SCALP (НОВЫЙ — настоящий скальпер)
- **ТФ:** 5m основной, 1m триггер. **4h/1h НЕ читает для направления.**
- **Вход:** цена у 5m микро-края (≈20 свечей) ИЛИ 1m liquidity-sweep, **подтверждённый стаканом** (OBI/CVD в сторону входа). Обе стороны (long/short).
  - LONG: дип к 5m-поддержке + bid-перекос + CVD разворачивается вверх.
  - SHORT: зеркально (шорт на swap, ENABLE_FUTURES_EXECUTION=true).
- **Направление от:** 1m/5m структуры + поток. Не фейдит 1m-импульс.
- **Профиль:** SCALP.
- **Почему не дерётся с трендом:** читает микро-поток; если рынок прёт вверх — bid-перекос, скальп берёт лонги, а не шортит силу.

### 5.2 Range mean-reversion (РЕКЛАССИФИКАЦИЯ текущего «range»)
- **ТФ:** 1h коридор + 15m триггер.
- **Вход:** фейд края коридора — **ТОЛЬКО в подтверждённом боковике**: 4h И 1h не трендовые **И нет свежего пробоя** (цена не делает новых N-барных хаёв/лоёв). Это убивает шорты по пробою.
- **Профиль:** POSITION (среднее ведение, цель — mid коридора).
- **Открытый вопрос:** оставляем как отдельный движок или ретайрим, когда заработает настоящий скальп? (см. §10)

### 5.3 Trend continuation (волк)
- **ТФ:** 4h/1d биас + 15m вход (пуллбэк в тренд).
- **Вход:** согласие мульти-ТФ (есть), вход на откате.
- **Профиль:** POSITION. **Фикс:** сайзинг должен давать открываемую сделку (сейчас тренд-лонг резался экономикой при margin 350).
- **Почему важно:** когда рынок трендует (как сейчас вверх), ИМЕННО этот движок забирает движение — вместо того чтобы range его шортил.

### 5.4 CRT (A→M→D)
- **ТФ:** 4h-свеча (C1/C2) + 5m/15m вход (MSS/FVG). Уже построен.
- **Профиль:** POSITION (едем до противоположной ликвидности, TP1/TP2).
- Оставляем как есть, вписываем в общий каркас профилей.

## 6. Общий слой риска/исполнения (под всеми движками)

- **anti-drain:** маржа/экономика/лимит позиций. Пороги — из профиля (scalp мягче, position строже).
- **symbol-performance guard:** снижение риска/блок по серии убытков (работает, DOT/SOL заблокированы).
- **depth-гейт:** spread + OBI/стенки. Для скальпа — часть ВХОДА (не просто гейт). Для позиции — гейт качества.
- **post-loss cooldown:** оставить для scalp/range; для position не нужен (низкая частота).
- **Hard stop / TP1 / TP2:** у каждого профиля свои.

## 7. Оркестрация (per символ, per цикл)

```
Определяем режим символа по 4h/1h:
  ┌─ ТРЕНД (4h+1h согласованы) ──► TREND + CRT активны (в сторону тренда).
  │                                Range mean-revert ВЫКЛЮЧЕН (не фейдим тренд).
  ├─ БОКОВИК (подтверждён) ──────► Range mean-revert + CRT-развороты активны.
  │                                Trend off.
  └─ во ВСЕХ режимах ───────────► micro-flow SCALP работает на LTF независимо
                                   (он TF-изолирован; идёт по 1m/5m потоку).
```

Так «не драться с шерстью» — это **свойство архитектуры**, а не фильтр: range-фейд физически не запускается в тренде; скальп читает поток и сам не шортит ралли.

Одна позиция на символ остаётся (приоритет: trend > CRT > range; scalp — отдельная книга/слот, если решим разводить, см. §10).

## 8. Консолидация конфига

**В код (структурированные профили, один источник правды):**
```python
ENTRY_ENGINES = {
  "scalp":  {tfs:["1m","5m"], target_pct:0.4, stop_pct:0.4, ...},
  "range":  {tfs:["1h","15m"], require_confirmed_range:true, ...},
  "trend":  {tfs:["4h","15m"], ...},
  "crt":    {tfs:["4h","5m"], rr_tp2:2.0, ...},
}
MGMT_PROFILES = {
  "scalp":    {arm_pct:0.2, breakeven_fast:true, cvd_exit:true, time_stop_min:N, size_pct:0.10},
  "position": {ride:true, trail_dd:0.5, cvd_force_exit:false, size_by:"conviction"},
}
```

**Остаётся в env (инфра/секреты/бизнес):**
`HTX_*`, `DATABASE_URL`, `REDIS_URL`, `CORS_ORIGINS`, `TELEGRAM_*`, `VIP_*`, `OWNER_*`, `JWT_SECRET`,
`RISK_EQUITY_USDT`, `ROBOT_MODE`, `ENABLE_<strategy>` (ON/OFF), `ENABLE_LIVE_ORDERS`.

**Убираем из env** (уезжают в профили): `OB_CVD_*`, `SCALP_*`, `POST_LOSS_*`, `ANTI_DRAIN_*` (кроме глобального équity), `PROD_GATE_*` (в профиль качества), `SYMBOL_PERF_*` (в guard-конфиг). ~20 переменных → 0.

## 9. План миграции (фазы, каждая отдельно тестируется на paper)

1. **Консолидация конфига** в профили (поведение НЕ меняем) — чистая база, убираем кашу.
2. **Range → confirmed-range only** (гейт по пробою/тренду) — сразу прекращает шорты по пробою.
3. **Fix trend POSITION sizing** — трендовые лонги начинают открываться (ловим движение).
4. **Build micro-flow SCALP** (1m/5m + стакан, обе стороны) — настоящий скальпер.
5. **Per-profile management** — scalp (тугой/тайм-стоп) vs position (ride) разведены чисто.
6. **Валидация** каждой фазы по `missed_profit`, winrate, net, по типам.
