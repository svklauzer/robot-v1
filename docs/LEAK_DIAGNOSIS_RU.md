## Первоисточники убытка (по приоритету)

### Leak 1 — Вход в неблагоприятный поток (главный)
`orderbook_analyzer.py:143` — гейт лонга:
```
if sig.obi < obi_confirm and sig.bid_wall_share < wall_confirm: return False
```
Блокировка требует **И** слабого OBI, **И** слабой стенки. Достаточно стоячей
бид-стенки ≥0.30, чтобы лонг прошёл при OBI −0.5…−0.68 (агрессивные продажи).
Стенку легко «съедают» / спуфят → ложная опора.

Жёсткое вето (`OB_OBI_HARD_VETO=0.75`) срабатывает только на экстриме −0.75;
зона −0.5…−0.68 (уже сильное давление против) проходит.

CVD-фильтр входа (`OB_CVD_ENTRY_BLOCK_RATIO=0.6`) требует `cvd_trades ≥ 25`
за 60с (`OB_CVD_MIN_TRADES=25`). В живом стакане тонких символов 1–5 сделок →
фильтр **никогда не включается** именно там, где он нужен.

Доказательство (лонги, все убыточны):
- #198 XRP: obi −0.681, cvd_ratio −1.0 (100% продажи), cvd_trades 1 → вход → −0.31
- #194 ADA: obi −0.672 → −0.84
- #197 AVAX: obi −0.499 → −0.54
- #191 XRP: obi −0.530, cvd −60.9 → −0.83

### Leak 2 — Контртрендовые CRT-лонги в медвежьей ленте
`crt_strategy.py:116` — trend-align режет лонг только если строка ТФ == `trend_down`.
Когда ТФ читаются как `mixed`/`flat` (а моментум RSI/MACD медвежий) — лонг проходит.
Скан прямо сейчас показывает `watch_short` по большинству символов, BTC 1h RSI 16,
ETH/AVAX/ADA momentum bearish — а движок продолжает публиковать `crt_bull_sweep_CRL`
лонги. Это фейд силы → ловля стопа.

### Leak 3 — Нет рабочей фиксации прибыли (порог защиты выше реального MFE)
Активные значения (.env):
```
MIN_PROTECTIVE_EXIT_PCT = 1.80      MFE_CAPTURE_START_PCT = 0.90
MIN_PROTECTIVE_NET_USDT = 2.50      TREND_RIDE_MIN_MFE_TO_PROTECT_PCT = 1.2
```
Реальный MFE этих сетапов — **0.1…0.9%**. Значит protective / trail / capture
**не вооружаются никогда**. Единственное, что может закрыть плюсовую сделку —
хард-стоп на −1%. Поэтому 77.8% сделок едут из зелёного в полный стоп.
Пример SOL #199: MFE +0.126%, стоп −1.04% → −0.56 USDT, missed_profit 1.21%.

### Leak 4 — «breakeven_lock» закрывается в минус
`exit_policy.py` (блок breakeven_lock): вооружается при MFE≥0.80%, но условие
выхода `current_pct ≤ be_floor(0.10%)` при `soft_exit_confirmed` (нужен flow_against
= 15 сделок CVD) ИЛИ `current_pct ≤ −0.35%`. В тонком стакане поток не
подтверждается → замок ждёт −0.35% и исполняется по рынку уже на −0.4…−0.55%.
Это **не безубыток, а отложенный стоп**:
- SOL #186 breakeven_lock → −0.56% / −1.85 USDT
- ADA #194 breakeven_lock → −0.54% / −0.84 USDT
- AVAX #196 breakeven_lock → −0.52% / −0.52 USDT

### Leak 5 — Асимметрия стоп/MFE и ML-гейт
Стопы ~1% при MFE ~0.2–0.9% → отрицательный реализованный RR структурно.
ML meta-labeler AUC 0.70, но `full_auto` с порогом входа 0.45. Калибровка shadow:
бакет score 0.30–0.45 → winrate 10% (Net PnL −11.85), а гейт «score<0.45» сэкономил
бы +14.1 USDT. Многие убыточные проходили со score 0.49–0.55 — порог слишком низкий.

### Leak 6 — Грид тоже кровит
`grid/state`: realized −1.56 USDT, свежие циклы: BTC `grid_stop_loss` −0.86,
SOL −0.58, ETH −1.11. Гриды ловят стопы в том же чопе на 1h.

## План устранения (по эффекту)

**A. Вход — закрыть дыру потока (приоритет 1).**
1. `orderbook_analyzer.py:143`: заменить `and` на `or` ЛИБО не «спасать» стенкой
   при отрицательном OBI: блок, если `obi < obi_confirm` и (стенка тонкая **или**
   `obi ≤ −0.35`). Т.е. стоячая стенка не должна перебивать активный поток против.
2. Снизить `OB_OBI_HARD_VETO` 0.75 → **0.45** (не лонговать при OBI ≤ −0.45).
3. CVD-фильтр входа должен работать на тонкой выборке: понизить `OB_CVD_MIN_TRADES`
   для входа до 3–5 **или** при `cvd_trades < N` и `cvd_ratio ≤ −0.9` считать это
   «нет опоры» и блокировать (сейчас тонкая выборка = пропуск).

**B. Тренд-фильтр (приоритет 1).**
4. `crt_strategy.py:116`: расширить блок контртренда — резать лонг не только при
   `trend_down`, но и при медвежьем моментуме (RSI<45 и MACD_hist<0 на 15m/1h),
   и наоборот для шорта. Согласовать со скан-состоянием `watch_short/watch_long`.

**C. Выход — включить реальную фиксацию (приоритет 1).**
5. Опустить порог защиты под фактический MFE: `MIN_PROTECTIVE_EXIT_PCT` 1.80 → **0.5–0.7**,
   `MFE_CAPTURE_START_PCT` 0.90 → **0.4**, `TREND_RIDE_MIN_MFE_TO_PROTECT_PCT` 1.2 → **0.5**.
   Цель — фиксировать, когда сделка реально бывает в плюсе.
6. Починить breakeven_lock: выход при `current_pct ≤ 0` (истинный безубыток),
   не ждать flow-подтверждения в тонком стакане; убрать зависимость от −0.35%.
7. Подтянуть стоп ближе к структуре / MFE: если сетап даёт MFE ~0.5%, стоп −1% —
   это RR < 1 по факту. Пересчитать `CRT_STOP_BUFFER` и размер под реальный MFE.

**D. Размер / ML.**
8. Поднять ML-гейт входа 0.45 → **0.55–0.60** (по shadow-калибровке это плюс).
9. `RISK_PER_TRADE_PCT=0.4` оставить, но включить kill_switch (сейчас disabled).

**E. Грид.** Заморозить открытие новых грид-циклов, пока чоп/медвежий режим
(эконом-фильтр открытия + regime-band), либо только со стороной тренда.
